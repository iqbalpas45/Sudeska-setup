🎯 Tujuan Sistem

Sistem SUDESKA adalah platform automasi end-to-end yang mengelola seluruh siklus kerja supervisi penandaan batas areal Perhutanan Sosial, mulai dari penerimaan permohonan, perencanaan instruksi kerja berbasis GIS, pengumpulan data lapangan via KoboToolbox, hingga penerbitan Berita Acara, peta, dan data spasial final — seluruhnya dikendalikan melalui satu portal berbasis form & generator.

TAHAP -2 Penerimaan Permohonan

Input

  • File line areal kerja (.shp / .kml / .geojson)
  • File polygon permohonan (.shp / .kml)
  • Nama KTH / pemohon
  • Luas areal (Ha)
  • Kabupaten / Provinsi
  • Nomor SK persetujuan
  • Tanggal permohonan

Proses

  • Validasi kelengkapan berkas
  • Validasi geometri (not null, valid polygon)
  • Assign ID_PERMOHONAN unik
  • Registrasi ke sistem

Output

  • Nomor registrasi permohonan
  • Paket berkas digital tervalidasi
  • Notifikasi status ke pemohon
Variabel KunciFormatWajib?Catatan
ID_PERMOHONANText (auto)YaFormat: BPKH-XI-{TAHUN}-{SEQ}
nama_kthTextYaSesuai SK persetujuan
nomor_skTextYaSK MENLHK persetujuan
luas_haDecimalYaDari SK; dicek vs hitung ulang GIS
geom_lineLineString (UTM)YaBatas areal kerja
geom_polygonPolygon (UTM)YaAreal permohonan
TAHAP -1 Analisis Segmentasi Batas

Input

  • Geometri permohonan (line + polygon)
  • Base data jalan
  • Base data sungai
  • Layer batas kawasan hutan
  • Layer APL
  • Layer areal izin lain (PPHKm, PPHD, PPKH, dll)
  • Parameter buffer (misal jalan: 10m, sungai: 20m)

Proses

  • Normalisasi projeksi UTM
  • Overlay spatial (buffer + intersect)
  • Split line di tiap perubahan overlay
  • Klasifikasi segmen per prioritas
  • Assign atribut tipe segmen

Output

  • Layer segmen terklasifikasi (shapefile)
  • Rekap panjang per tipe segmen
  • Peta preview segmentasi (PNG/PDF)
Tipe SegmenAturan KlasifikasiPrioritas
Batas AlamDalam buffer jalan/sungai1 (tertinggi)
Batas LuarBerbatasan kawasan hutan vs APL2
Batas SekutuBerbatasan areal izin lain3
Batas SendiriTidak memenuhi kriteria lain4 (default)
TAHAP 0 Generasi Trayek & Titik Batas

Input

  • Layer segmen terklasifikasi
  • Aturan interval titik per tipe segmen
  • Titik ikat / titik kontrol
  • Zona UTM

Proses

  • Generate titik batas per aturan interval
  • Urutan trayek (from→to searah jarum jam)
  • Hitung azimuth & jarak datar tiap trayek
  • QA/QC topologi & urutan
  • Export CSV master instruksi

Output

  • Layer titik_batas_instruksi (shapefile)
  • Layer trayek_batas (polyline)
  • CSV master instruksi (No, Nama, X, Y, Keterangan)
  • Tabel trayek (from, to, azimuth, jarak)
TAHAP 1 Generator Instruksi Kerja

Input

  • Hasil tahap 0 (CSV master + geodata)
  • Nomor instruksi kerja
  • Tanggal instruksi
  • Nama pejabat penandatangan
  • Panjang batas (m), Luas (Ha)
  • Detail titik kontrol / titik ikat
  • Template DOCX instruksi kerja

Proses

  • Isi placeholder template DOCX
  • Generate tabel trayek + tabel titik instruksi
  • Render layout peta instruksi
  • Generate XLSForm Kobo dari CSV master

Output

  • 📄 DOCX Instruksi Kerja final
  • 🗺️ PDF Peta Instruksi (layout baku)
  • 📍 Geodata instruksi (point/line/polygon)
  • 📊 XLSForm Kobo siap upload
TAHAP 3 Supervisi via KoboToolbox

Input

  • XLSForm Kobo (dari Tahap 1 atau generator HTML)
  • Data lapangan petugas supervisi (GPS / manual)
  • Foto dokumentasi
  • Catatan lapangan
  • Status validasi per titik

Proses

  • Kobo collect data per titik PAL
  • Inline lookup X/Y instruksi (tanpa media CSV)
  • Hitung jarak deviasi instruksi vs pelaksanaan
  • Tentukan status: Valid / Cek Validasi / Tidak Ditemukan
  • Ekspor rekap tabel gabungan

Output

  • 📊 Tabel gabungan per titik (Instruksi + Pelaksanaan + Supervisi)
  • 📊 XLSX rekap validasi
  • 📋 Ringkasan status validasi seluruh titik

TAHAP 4 Generator Berita Acara + Output Final

Input

  • Tabel gabungan supervisi (Tahap 3)
  • Nomor & tanggal BA
  • Identitas tim supervisi (nama, NIP, jabatan)
  • Identitas pendamping / pemohon
  • Titik ikat + koordinat acuan
  • Luas hasil + panjang batas hasil
  • Rincian tanda batas terpasang
  • Narasi toleransi & kesimpulan
  • Nama pejabat penandatangan & jabatan

Proses

  • Isi template BA DOCX (placeholder otomatis)
  • Generate tabel gabungan instruksi-pelaksanaan-supervisi
  • Render layout peta supervisi final
  • Ekspor geodata final
  • Paketkan semua output ke ZIP

Output

  • 📄 DOCX Berita Acara Supervisi
  • 🗺️ PDF Peta Supervisi (layout baku)
  • 📍 Shapefile final (point/line/polygon)
  • 📦 ZIP paket arsip digital kegiatan

⚙️ Tech Stack per Modul

📄 Generator DOCX

  • docxtemplater (JS library)
  • PizZip (untuk buka .docx template)
  • FileSaver.js (download)
  • Template .docx dengan placeholder {field}

📊 Generator XLSX / XLSForm

  • SheetJS xlsx@0.18.5 (sudah ada)
  • FileSaver.js (sudah ada)
  • Pure JavaScript

🗺️ Mesin Peta & Geodata

  • Leaflet.js (preview peta interaktif)
  • Turf.js (kalkulasi spasial: buffer, intersect, distance)
  • shpwrite (export Shapefile dari JS)
  • atau: ArcGIS untuk peta formal

🖨️ PDF Output

  • Jika via HTML: html2canvas + jsPDF
  • Jika peta formal: ArcGIS layout export
  • Print CSS sebagai fallback cepat

📦 Packaging

  • JSZip (buat file ZIP output)
  • FileSaver.js (download ZIP)

🌐 Hosting & Deployment

  • GitHub Pages (tanpa server)
  • Pure client-side HTML/JS/CSS
  • Tidak butuh backend / database

📱 KoboToolbox

  • XLSForm (XLSX) sebagai definisi form
  • KoboCollect (Android) untuk lapangan
  • Export CSV/XLSX hasil submit

🗂️ Format Data Spasial

  • GeoJSON (transfer antar modul)
  • Shapefile ZIP (output formal)
  • CSV koordinat (kompatibel ArcGIS/QGIS)
  • KML (opsional, untuk Google Earth)

⚠️ Daftar Risiko & Mitigasi
NoRisikoDampakProbabilitasMitigasi
1 Mesin GIS segmentasi mengklasifikasi segmen salah Tinggi – titik instruksi keliru, data lapangan salah Sedang QA/QC manual wajib sebelum generate instruksi; tambah step review visual peta segmen
2 ID_TITIK tidak konsisten antar modul Tinggi – tabel gabungan supervisi tidak bisa di-join Sedang Paksa ID_TITIK dari Tahap 0, beku di seluruh siklus, tidak boleh diedit manual
3 Koordinat GPS lapangan deviasi besar Sedang – status validasi "Cek Validasi" menumpuk Tinggi Tetapkan ambang toleransi jelas; tambah kolom alasan override di rekap
4 Template DOCX berubah tanpa sinkronisasi placeholder Sedang – generator DOCX error atau output berantakan Rendah Versi template DOCX terkontrol; changelog placeholder wajib
5 Browser tidak mendukung library JS tertentu Sedang – form tidak bisa generate output Rendah Test di Chrome/Edge/Firefox sebelum deploy; tambah pesan error jelas ke user
6 Export Kobo format berubah Rendah – parser rekap perlu update Rendah Normalkan header kolom Kobo saat import; tambah step mapping header
7 File geometri upload rusak / format salah Rendah – modul tidak bisa proses Sedang Validasi format & geometri saat upload; tampilkan pesan error spesifik
